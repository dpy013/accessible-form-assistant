# 🚀 信息无障碍表格填写助手 - 最终开发方案

## 👤 角色设定

你是一位拥有10 年经验的Python 桌面应用架构师，精通wxPython、python-docx、Pillow 及软件构建。你擅长构建高可用、具备”数据闭环”能力的无障碍测试辅助平台，并坚持全虚拟环境开发规范。

## 🎯 项目目标

开发“信息无障碍表格填写助手”。该工具旨在通过极简的交互（选择场景/模板 -> 自动编号 -> 填报）实现无障碍测试的高效管理。 核心亮点：
1. 极简工程创建：无需输入名称，仅需选择场景/模板，系统自动生成编号（如 #0404）并创建文件夹。
2. 场景覆盖全面：支持 Windows、Web、App、小程序等多端场景。
3. 多格式导出：支持 HTML、Word、Excel、CSV、Markdown 等多种格式导出。
4. 工程物理隔离：每个编号对应一个独立文件夹，资源与数据分离。
5. 全虚拟环境：开发、测试、构建全流程在 uv 虚拟环境中进行。

## 🛠️ 1. 技术栈
- 语言：Python 3.14.3
- 包管理：uv (虚拟环境)
- GUI：wxPython (wx.richtext, wx.dataview)
- 数据处理：python-docx, openpyxl, json, Pillow
- HTML 导出：jinja2 (HTML 模板)
- 构建：scons

## 📦 2. 核心功能模块（最终版）

### 2.1 智能工程向导（自动编号 & 无名称模式）

- 创建流程：
  - 步骤一：选择场景（下拉框选项）：
    - Windows (新增)
    - Web
    - App (Android/iOS)
    - 小程序
  - 步骤二：选择模板（根据场景动态加载，如 WCAG 2.1, GB/T 37668, Windows App 	Accessibility Guidelines）。

  - 步骤三：自动初始化（无需用户输入名称）。

- 自动编号逻辑：
  - 系统获取当前日期（月+日），生成编号 #MMDD。

  - 示例：4 月4 日创建 -> 编号 #0404。

- 物理目录生成：
  - 直接以编号命名文件夹：#0404/。

  - 内部结构：#0404/assets/ (截图), #0404/backup/ (备份), #0404/project.json。

### 2.2 交互式填报与资源管理

- 主列表：
  - 专注模式：工具栏增加”隐藏已完成”复选框，过滤掉”通过”项。
  - 回收站：支持软删除与还原。

- 富文本编辑器：
  - 截图物理化：Ctrl+V 粘贴 -> 压缩 -> 保存至 assets/ -> 插入相对路径。

  - AI 润色：调用 Prompt 生成标准缺陷描述。

### 2.3 多格式导出（增强版）

- 支持格式：
  - Word (.docx)：标准文档格式，支持双向同步（CustomXML）。

  - Excel (.xlsx)：表格格式，便于数据整理。

  - HTML (.html)：网页格式，支持交互式查看。

  - CSV (.csv)：表格文本格式，便于数据交换与工单导入。
  - Markdown (.md)：轻量文本格式，便于版本管理与协作。

- 导出功能：
  - HTML 导出：使用Jinja2 模板生成响应式HTML 页面，包含完整的测试报告内容。

  - CSV 导出：支持通用 CSV 数据导出。
  - Markdown 导出：支持 Markdown 文本导出。

### 2.4 数据持久化与容错

- 本地资源库：JSON 仅存相对路径，确保文件夹可随意迁移。

- 自动备份：每 5 分钟将 project.json 复制至 backup/。
